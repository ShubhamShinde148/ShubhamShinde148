## <img src="https://media.giphy.com/media/hvRJCLFzcasrR4ia7z/giphy.gif" width="30"/> 🏆 [CONSTRAINTS SOLVED] ~ CTF Writeups & Solutions

Below is a structured log of advanced CTF challenges solved across various platforms (SpaceCTF, ZeroCTF, G.H.O.S.T CTF), showcasing mathematical models, vulnerability chains, and automated solve scripting.

<details>
<summary><b>🗝️ Cryptography Challenges</b></summary>
<br/>

*   **Twice-Told Secret (Franklin-Reiter Related Message Attack)**
    *   *Category/Diff:* Cryptography | Hard
    *   *Concept:* RSA linear relation $m_2 \equiv m_1 + r \pmod n$ with exponent $e = 3$.
    *   *Exploitation:* Implemented Euclidean polynomial division on $f_1(x) = x^3 - c_1 \pmod n$ and $f_2(x) = (x+r)^3 - c_2 \pmod n$ to find the common linear root.
    *   *Solve Script:* [`solve_twice_told.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/solve_twice_told.py)
*   **The Crow's Parity (RSA LSB Parity Oracle)**
    *   *Category/Diff:* Cryptography | Medium–Hard
    *   *Concept:* Leaking the Least Significant Bit of the plaintext on decryption.
    *   *Exploitation:* Built binary-search interval narrowing logic, pre-computing multiplier ciphertexts $c_i \equiv c \cdot (2^i)^e \pmod n$ and querying them concurrently via pipelined streams.
    *   *Solve Script:* [`solve_parity.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/solve_parity.py)
*   **Wall of Wildlings (Modular Hill Cipher)**
    *   *Category/Diff:* Cryptography | Medium
    *   *Concept:* 3x3 matrix Hill Cipher encryption mapping plaintext blocks: $C = P \cdot K \pmod{26}$.
    *   *Exploitation:* Extracted modular inverse key matrix using modular determinants and adjugate matrices: $K^{-1} = C^{-1} \cdot P \pmod{26}$.
    *   *Solve Script:* [`solve56.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/solve56.py)
*   **Lost Signal (Visual Pixel Steganography)**
    *   *Category/Diff:* Crypto/Stego | Medium
    *   *Concept:* Difference-based bit hiding between baseline and transmission PNG frames.
    *   *Exploitation:* Extracted raw RGB variances, corrected boundary conditions (e.g. brightness clashes at $x=80$), and reconstructed the ASCII payload.

</details>

<details>
<summary><b>⚙️ Reverse Engineering Challenges</b></summary>
<br/>

*   **Masterpiece of P!3t (Esoteric Piet Image Tracing & 3D Parsing)**
    *   *Category/Diff:* Reverse Engineering | Hard
    *   *Concept:* Code logic hidden inside 2D Piet image codel color boundaries.
    *   *Exploitation:* Wrote an image parser to trace alternating RGB block widths, decapsulated a Base64-encoded GZIP stream of `flag.obj`, and plotted coordinates to visually reconstruct the 3D model flag.
    *   *Solve Scripts:* [`extract_full_payload.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/extract_full_payload.py) & [`plot_3d_obj.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/plot_3d_obj.py)
*   **Greywind's Trail (Modular Hash Inversion)**
    *   *Category/Diff:* Reverse Engineering | Medium
    *   *Concept:* Custom hashing algorithm with modular accumulation: $A_i = (A_{i-1} \cdot \text{0x41c64e6d} + C_i \cdot \text{0x3039}) \pmod{2^{32}}$.
    *   *Exploitation:* Reversed the linear congruence relation using the modular multiplicative inverse of $\text{0x3039} \pmod{2^{32}}$ to reconstruct the input bytes.
    *   *Solve Script:* [`solve_greywind.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/solve_greywind.py)
*   **Space VM Interpreter**
    *   *Category/Diff:* Reverse Engineering | Hard
    *   *Concept:* Virtual Machine emulator running customized bytecode registers.
    *   *Exploitation:* Traced emulator states, stack operations, and instructions to retrieve constraints on stack register memory to unlock the flag.
*   **Ned's Last Riddle (Prefix Sum Inversion)**
    *   *Category/Diff:* Reverse Engineering | Easy–Medium
    *   *Concept:* Dynamic prefix sum accumulators modulo 256.
    *   *Exploitation:* Reversed prefix sums using adjacent subtraction constraints: $R[i] = (Target[i] - Target[i-1]) \bmod 256$.

</details>

<details>
<summary><b>🌐 Web & Exploit Chains</b></summary>
<br/>

*   **The Three-Eyed Raven (Full-Chain Exploitation)**
    *   *Category/Diff:* Web/Exploits | Hard
    *   *Vulnerability Chain:* 
        1. SQL Injection (`OR 1=1 --`) to leak JWT parameters.
        2. JWT RS256 to HS256 algorithm confusion signing forgery using the public key as the HMAC secret.
        3. Stack Buffer Overflow in execution binary to hijack return pointer and force code redirection.
    *   *Solve Script:* [`solve_raven.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/solve_raven.py)
*   **G.H.O.S.T. Upload Secure Sandbox (File Upload & Path Traversal)**
    *   *Category/Diff:* Web | Medium
    *   *Concept:* File upload filter bypass combined with LFI.
    *   *Exploitation:* Crafted JPEG header magic bytes (`\xFF\xD8\xFF\xE0`) ahead of PHP webshell payload to pass MIME checks, bypassed nested LFI filters (`....//`) to traverse paths, and executed remote shell commands.
*   **Faceless Verifier (JWT Algorithm Confusion)**
    *   *Category/Diff:* Web/Crypto | Medium
    *   *Concept:* RS256 algorithm fallback vulnerability.
    *   *Exploitation:* Signed unauthorized admin tokens using the server's public key (complete with trailing newline formatting) as the sym-key HMAC secret.
    *   *Solve Script:* [`solve_faceless_token.py`](file:///C:/Users/shubh/Documents/antigravity/jolly-archimedes/solve_faceless_token.py)

</details>

<details>
<summary><b>⛓️ Blockchain & Smart Contract Challenges</b></summary>
<br/>

*   **Dragonglass Mint (Signature Replay Attack)**
    *   *Category/Diff:* Blockchain | Hard
    *   *Concept:* Minting logic lack of signature nonces or tracking mechanisms.
    *   *Exploitation:* Replayed verified minting signatures. Built an exploit payload utilizing **TCP Pipelining** to bypass network round-trip overhead, executing 10,000 mint transactions simultaneously before session timeout.
*   **Faceless Vault (EVM Storage Collision)**
    *   *Category/Diff:* Blockchain | Medium
    *   *Concept:* EVM storage layout delegation conflicts.
    *   *Exploitation:* Triggered `delegatecall` to a library writing directly to Slot 0 (`librarian`), which collided with Slot 0 (`owner`) of the main Vault contract, transferring owner access to the caller.

</details>
